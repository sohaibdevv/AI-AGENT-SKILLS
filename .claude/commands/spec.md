Load skills/spec-driven-development/SKILL.md and skills/requirements-analysis/SKILL.md.

Write a specification for the feature or change described. Follow the spec-driven-development process exactly:
1. Capture the raw requirement
2. Identify stakeholders and success criteria
3. Write functional requirements (each independently testable)
4. Write non-functional requirements
5. Define the data model
6. Define the interface contract
7. List all open questions

Do not write any implementation code until the spec is complete and all open questions are resolved.
