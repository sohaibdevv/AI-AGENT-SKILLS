Load skills/test-driven-development/SKILL.md and agents/test-engineer.md.

Write or review tests for the code described. Ensure:

1. **Happy path** - Core behavior tested
2. **Error paths** - Every error condition tested
3. **Edge cases** - Empty, null, boundary values, concurrent access
4. **Test names** - Describe expected behavior, not implementation
5. **Independence** - Each test runs in isolation, order doesn't matter
6. **Speed** - Tests must run in under 5 minutes for the local suite

For each missing test: describe what behavior it should cover and why it matters.
For each poor test: explain what's wrong and provide an improved version.
