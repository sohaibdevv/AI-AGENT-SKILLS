Create a task breakdown for the work described. For each task:
- Format: [Verb] [Object] [Constraint]
- Estimate: S (< 4 hours) / M (< 2 days) / L (> 2 days — break it down further)
- Done criteria: specific, observable "done" state

Identify: dependencies between tasks, the critical path, any tasks estimated > 2 days (break them down), and the riskiest tasks (sequence first).

If architectural decisions need to be made: write ADRs for decisions that are hard to reverse.
