# Frontend Pack

Load this pack when building web UIs - components, pages, or design systems.

## Skills to load

1. `skills/spec-driven-development/SKILL.md` - Define all UI states before building
2. `skills/frontend-engineering/SKILL.md` - Performance, accessibility, testing
3. `skills/accessibility-engineering/SKILL.md` - WCAG 2.2, keyboard, screen reader
4. `skills/performance-optimization/SKILL.md` - Measure before optimizing
5. `skills/security-and-hardening/SKILL.md` - XSS, CSP, CSRF
6. `skills/test-driven-development/SKILL.md` - Component and E2E tests

## Personas
- `agents/code-reviewer.md` - For component PR reviews
- `agents/performance-engineer.md` - For perf-sensitive pages
- `agents/tech-writer.md` - For component documentation

## Frontend-specific checklist
- [ ] All UI states: loading, empty, error, populated
- [ ] Keyboard navigable
- [ ] Screen reader tested
- [ ] Lighthouse performance score ≥ 90
- [ ] No XSS vulnerabilities
- [ ] Responsive across breakpoints
- [ ] Error boundaries in place
